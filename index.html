준비물 설치 — 아직 아무것도 안 깐 분

5/28에 이미 세팅한 분은 건너뛰고 STEP 01로. 처음이면 위에서부터 4개를 설치하세요. 단계마다 방식 태그를 붙여뒀습니다 —🌐 홈페이지 는 사이트에서 받아 더블클릭, 🧩 VS Code 확장 은 VS Code 안에서 검색·설치, ⌨ 터미널 은 명령어 붙여넣기.

1
VS Code = 작업 화면(에디터) 🌐 홈페이지
오늘 모든 작업이 이 안에서 일어납니다. 공식 사이트에서 본인 OS 버튼을 눌러 받고 기본값(Next만 계속)으로 설치.
다운로드 ↓  code.visualstudio.com/download  (Windows / Mac 버튼 선택)
2
한글 언어팩 = 메뉴 한글화 (선택) 🧩 VS Code 확장
영어 UI가 부담되면 설치. VS Code 왼쪽의 확장(Extensions) 아이콘 → 검색창에 Korean Language Pack → 게시자 Microsoft 것 Install → 안내 뜨면 재시작. 메뉴가 한글로 바뀝니다.
👀 검색 결과 맨 위 "한국어 언어 팩 / Korean Language Pack for Visual Studio Code" (Microsoft)
3
Git = 파일 이력·발행 도구 🌐 홈페이지
결과물을 저장·되돌리기·공유(발행)할 때 씁니다. 사이트에서 받아 기본값으로 설치(옵션 많아 보여도 전부 Next).
다운로드 ↓  Windows git-scm.com/download/win · Mac git-scm.com/download/mac  (brew 있으면 brew install git)

STEP 02 · 다 같이

온보딩 — 책상 주고 업무 매뉴얼 알려주기

입력은 전부 한국어 문장. 명령어 문법 없음. 아래 4개를 잡으면 "매번 설명하는 AI"가 "내 업무를 아는 직원"이 됩니다.

1
프로젝트 폴더 = 담당 구역 / 책상
클라이언트·업무별 폴더 1개가 작업 단위. 그 폴더를 열면 그게 맥락(cwd)이 됩니다. 이후 "이 폴더 안에서" "방금 그거"로 가리킴.
입력: ○○클라이언트 폴더 만들고 그 안에서 작업할게
2
CLAUDE.md = 업무 매뉴얼 · 온보딩 문서
이 직원이 알아야 할 톤·금지·구조·담당을 적어두는 파일. 한 번 적으면 매번 안 알려줘도 그 맥락으로 일합니다. 입력: 이 폴더 CLAUDE.md 만들어줘 하면 질문에 답만 하면 됩니다.
CLAUDE.md — 예시
# 나: 김OO · 콘텐츠팀 · 담당 ○○치과
# 톤: 담백하게, 과장 금지 · 사실 기반(모르면 "확인 필요")
# 보고서는 docx, 공유는 HTML 링크
# 폴더: /리서치 /초안 /발행
3
커맨드 = 자주 시키는 일 매뉴얼화
잘 된 작업을 /이름 한 줄로 박제. 이미 깔린 것: /리서치 /이어서저장 /박제 /사용진단. 내 것 추가:
입력: 매주 하는 운영보고 작업을 /주간보고 커맨드로 만들어줘
4
에이전트 = 전문 분야 팀원 충원
특정 역할만 맡는 보조(리서치 전담·검수 전담). 여러 명이 병렬로 분담. 기본 제공 + 내 업무용 추가:
입력: 경쟁사 리서치 전담 에이전트 만들어줘 → .claude/agents/에 역할 정의로 저장됨
👀 실제로는 이렇게 한국어 대화로 흘러갑니다 (예시 — 본인 업무에 맞게):
claude — 온보딩 대화
> ○○치과 폴더 만들고 그 안에서 작업할게
⏺ ~/HIZ/○○치과 만들었어요. 지금부터 이 폴더가 작업 맥락(cwd)입니다.

> 이 폴더 CLAUDE.md 만들어줘
⏺ 몇 가지만 알려주세요 — 담당 업무? 톤? 보고서 형식?
> 치과 블로그 콘텐츠 담당, 담백한 톤, 보고서는 docx
⏺ CLAUDE.md 저장했어요. 다음부터 이 폴더에선 안 알려줘도 이 맥락으로 일합니다.
온보딩 팁한 번에 완벽할 필요 없음. 대충 시키고 "아니 그거 말고~"로 고쳐나가기. 신입 가르치듯, 한 번 알려준 건 CLAUDE.md에 적어두면 다음부터 안 까먹습니다.
STEP 03

도구·교육 — 스킬(교육)과 MCP(도구 연결)

두 가지만 구분하면 됩니다 — 스킬(클로드의 능력)과 MCP(외부 도구·계정 연결).

스킬과 MCP 차이
  • 🎓 스킬 = 노하우·능력 교육. 브레인스토밍·디자인·문서 만들기 같은 "이 작업은 이렇게" 기법. 외부 계정 없이 설치만 하면 됩니다.
  • 🔌 MCP = 외부 도구·계정 연결. 이미지 생성기·Notion·Slack처럼 클로드를 바깥 서비스에 접속. 키·로그인이 필요합니다.

🎓 스킬 — 클로드의 능력 (자동 탑재, 계정 불필요)

seed에 기본 탑재돼 설치하면 바로. 관련 작업을 시키면 자동으로 켜집니다.

기능무엇 / 어떻게
기획·브레인스토밍아이디어 발산 → 설계 정리. "이 기획 같이 잡자" (superpowers)
웹·시안 디자인보기 좋은 HTML·페이지 생성 (frontend-design)
문서 생성"보고서 pptx로" "이 표 xlsx로" — pdf·pptx·docx·xlsx
커맨드/리서치 /이어서저장 /박제 /사용진단
에이전트·규칙역할 보조 + HIZ 작업 규칙(톤·팩트원칙 등) 기본 적용
스킬 추가 방법
  • ① 이미 깔린 것 그냥 쓰기 — 위 기본 스킬은 관련 작업을 시키면 자동으로 켜집니다. "pdf로 빼줘" "pptx로"처럼 키워드만 말해도 활성.
  • ② 마켓플레이스에서 설치 — /plugin → 목록에서 방향키로 고르고 Enter.
  • ③ 직접 만들기 — 잘 된 작업을 "이 작업 스킬로 만들어줘" 또는 /박제. 반복 업무를 박아두면 다음부턴 한 마디로 끝.
💡 플러그인 = 스킬을 여러 개 담은 '상자'. /plugin에서 고르는 단위는 상자(플러그인)이고, 설치하면 안의 스킬들이 한꺼번에 들어옵니다. 예: superpowers 상자 하나 = 브레인스토밍·디자인 등 스킬 여러 개.

🔌 MCP — 외부 도구·계정 연결 (필요한 것만, 키 필요)

각자 키/로그인이 필요해 기본엔 안 들어갑니다. 필요한 것만 본인이 연결:

STEP 04

발행 — 결과물을 git으로 공유

결과를 터미널에 가둬두지 않고 링크 하나로 공유. 클로드가 대신 해주니 개념만 잡으면 됩니다.

GitHub(깃헙)이 뭐예요?작업물을 올려두는 온라인 저장소. 구글 드라이브와 비슷합니다. 다른 점은 — ① 되돌리기 이력(언제든 과거 시점으로) ② 웹 링크 자동 발급(HTML이면 페이지로 바로 열람) ③ 팀 공유. "결과물을 깃헙에 올린다"는 건 이 창고에 올려 공유 링크를 받는다는 뜻.
커밋 vs 푸시 — 이 둘만 구분하면 됨
  • 📸 커밋(commit) = 내 컴퓨터에 "중간저장". 지금 상태에 도장 찍기(되돌리기 지점). 아직 내 컴퓨터 안에만 있습니다.
  • ☁️ 푸시(push) = 그 저장본을 깃헙(온라인)으로 올려보내기. 이제야 링크로 공유 가능. (드라이브에 업로드하는 것과 같음)
  • 왜 둘로 나뉘나? 커밋은 작업하다 자주, 푸시는 공유할 때. 외울 필요 없습니다 — "깃헙에 올려줘" 한마디면 클로드가 둘 다 해요.
👀 명령어는 외울 필요 없이, 클로드에게 말로 시킵니다 (예시):
claude — 발행
> 이번 주 보고서 HTML로 빼서 깃헙에 올리고 링크 줘
⏺ 커밋하고 푸시할게요.
  [✓] git add .          (올릴 파일 담기)
  [✓] git commit -m "6월 3주차 운영보고"   (📸 중간저장)
  [✓] git push           (☁️ 깃헙에 올리기)
⏺ 푸시 완료 → github.com/hizsystem/내레포. 공개 레포면 HTML이 링크로 바로 열립니다.
🔑 처음 한 번 — GitHub 로그인처음 올릴 때 GitHub 로그인 창(브라우저)이 한 번 뜹니다. 회사 공유 GitHub 계정(hizsystem)으로 로그인하세요 — 계정 정보는 슬랙 DM. 한 번 하면 기억돼서 다음부턴 안 물어봅니다. (Claude 로그인과 같은 공유 계정 방식)
👀 푸시하면 깃헙(웹)에 이렇게 올라가 있습니다 (예시):
github.com/hizsystem/내레포
📁 hizsystem / 내레포 Public
📄 index.html6월 운영보고서방금
📄 report-2406.html6월 3주차 운영보고1일 전
📄 README.md첫 커밋3일 전
↑ 파일 옆 회색 글씨 = 내가 커밋할 때 단 메모(커밋 메시지). 이 화면에 보인다는 건 푸시로 깃헙에 올라왔다는 뜻.
🔗 HTML 파일은 hizsystem.github.io/내레포/ 주소로 바로 열람 — 이 링크를 클라이언트·팀에 공유.
공개 범위 주의링크 공유(웹 열람)는 공개 레포일 때. 클라이언트 기밀·내부 자료는 비공개 레포로. "이거 공개해도 되는 거야?"를 항상 확인.
