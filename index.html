온보딩 — 책상 주고 업무 매뉴얼 알려주기

입력은 전부 한국어 문장. 명령어 문법 없음. 아래 4개를 잡으면 "매번 설명하는 AI"가 "내 업무를 아는 직원"이 됩니다.

1
프로젝트 폴더 = 담당 구역 / 책상
클라이언트·업무별 폴더 1개가 작업 단위. 그 폴더를 열면 그게 맥락(cwd)이 됩니다. 이후 "이 폴더 안에서" "방금 그거"로 가리킴.
입력: ○○클라이언트 폴더 만들고 그 안에서 작업할게
2
CLAUDE.md = 업무 매뉴얼 · 온보딩 문서
이 직원이 알아야 할 톤·금지·구조·담당을 적어두는 파일. 한 번 적으면 매번 안 알려줘도 그 맥락으로 일합니다. 입력: 이 폴더 CLAUDE.md 만들어줘 하면 질문에 답만 하면 됩니다.
CLAUDE.md — 예시
# 나: 김OO · 콘텐츠팀 · 담당 ○○치과
# 톤: 담백하게, 과장 금지 · 사실 기반(모르면 "확인 필요")
# 보고서는 docx, 공유는 HTML 링크
# 폴더: /리서치 /초안 /발행
3
커맨드 = 자주 시키는 일 매뉴얼화
잘 된 작업을 /이름 한 줄로 박제. 이미 깔린 것: /리서치 /이어서저장 /박제 /사용진단. 내 것 추가:
입력: 매주 하는 운영보고 작업을 /주간보고 커맨드로 만들어줘
4
에이전트 = 전문 분야 팀원 충원
특정 역할만 맡는 보조(리서치 전담·검수 전담). 여러 명이 병렬로 분담. 기본 제공 + 내 업무용 추가:
입력: 경쟁사 리서치 전담 에이전트 만들어줘 → .claude/agents/에 역할 정의로 저장됨
👀 실제로는 이렇게 한국어 대화로 흘러갑니다 (예시 — 본인 업무에 맞게):
claude — 온보딩 대화
> ○○치과 폴더 만들고 그 안에서 작업할게
⏺ ~/HIZ/○○치과 만들었어요. 지금부터 이 폴더가 작업 맥락(cwd)입니다.

> 이 폴더 CLAUDE.md 만들어줘
⏺ 몇 가지만 알려주세요 — 담당 업무? 톤? 보고서 형식?
> 치과 블로그 콘텐츠 담당, 담백한 톤, 보고서는 docx
⏺ CLAUDE.md 저장했어요. 다음부터 이 폴더에선 안 알려줘도 이 맥락으로 일합니다.
온보딩 팁한 번에 완벽할 필요 없음. 대충 시키고 "아니 그거 말고~"로 고쳐나가기. 신입 가르치듯, 한 번 알려준 건 CLAUDE.md에 적어두면 다음부터 안 까먹습니다.

STEP 04

발행 — 결과물을 git으로 공유

결과를 터미널에 가둬두지 않고 링크 하나로 공유. 파일 안 던지고, 깃헙(GitHub)에 올리면 끝.

흐름은 3단계. 전부 "이거 깃헙에 올려줘" 한 마디면 클로드가 대신 칩니다:

git — 결과물 발행
$ git add .                       # ① 바뀐 파일 담기
$ git commit -m "6월 운영보고서"   # ② 스냅샷 저장(되돌리기 가능)
$ git push                        # ③ 깃헙으로 올리기 → 링크 공유
  • 커밋(commit) = 작업 중간저장. 망쳐도 이전 시점으로 되돌리기 가능 = 안전벨트.
  • 푸시(push) = 깃헙에 업로드. HTML이면 링크로 바로 열람 가능 → 클라이언트·팀 공유.
  • 입력 예: 이번 주 보고서 HTML로 빼서 깃헙에 올리고 링크 줘
👀 실제로는 한 마디면 클로드가 3단계를 대신 칩니다 (예시):
claude — 발행
> 이번 주 보고서 HTML로 빼서 깃헙에 올리고 링크 줘
⏺ 커밋하고 푸시할게요.
  [✓] git add .
  [✓] git commit -m "6월 3주차 운영보고"
  [✓] git push
⏺ 푸시 완료 → github.com/HIZ-Co/내레포. 공개 레포면 HTML이 링크로 바로 열립니다.
공개 범위 주의링크 공유(웹 열람)는 공개 레포일 때. 클라이언트 기밀·내부 자료는 비공개 레포로. "이거 공개해도 되는 거야?"를 항상 확인.
